# 设计文档

## 用户层

### CLI 结构

- 用户打开书的网页（如 `https://weread.qq.com/web/reader/7cb324e0727ab1f17cbf4c1`），书 ID 为 `7cb324e0727ab1f17cbf4c1`。
- **交互式模式**：`python -m weread_exporter_lys --interactive`，进入主页面 → 选择平台 → 输入书 ID → 选择输出格式 → 自动爬取。
- **命令行模式**：`--platform weread --url <url> --format md|txt|pdf|epub|mobi|azw3 [--headless]`。

架构上基于 `BookPlatform` 抽象基类，微信读书为已实现平台，预留扩展接口接入其他平台。

## 爬虫层（仅微信读书）

### 1. 初始化

用 Playwright 打开 `https://weread.qq.com/web/reader/$书ID`。等待登录（非 headless 时弹出扫码窗口），登录后保存 cookie/storageState 到 `cache/auth/weread-storage-state.json`。

### 2. 封面 & 目录

- 封面：从 `<meta og:image>` 或封面 img 取 URL，下载到 `cache/{书ID}/封面.jpg`。
- 目录：点击"目录"按钮，提取 `.readerCatalog_list_item`，保存为 `cache/{书ID}/toc.json`（`[{index, title, level}]`）。

### 3. 正文（WRPA 与非 WRPA）

**非 WRPA**：直接读 DOM 文本（`.readerChapterContent` 的 innerText），CSS 过滤掉 `.preRenderContainer` 等反爬垃圾。每章保存为 `content/{N}.md`。

**WRPA 加密**：正文通过 Canvas `fillText` 逐字绘制，DOM 中是 CSS 垃圾。通过 `hook.js` 在 Canvas `getContext('2d')` 前注入拦截：

- 拦截 `fillText(text, x, y)`，按 y 坐标分组成行，重组为文本流。
- 按 fontSize 区分标题（≥27px → `## `，≥23px → `### `）和正文。
- 过滤字体探测行（`abcdefghijklmn...`）。
- `getMarkdown()` 返回相邻去重后的各行拼接；`getLinesWithCoords()` 暴露带坐标的片段数组，供生僻字回插。

**章节内分页**：长章节（如《五帝本纪》）在桌面视口下按页拆分，每页 canvas 独立渲染、y 坐标独立。爬虫通过 `extract_full_chapter()` 自动检测"下一页"按钮并逐页翻页提取，每页独立采集生僻字 + 合 并，最后拼接（详见下节）。

### 4. 图片处理

`images.py` 的 `ImageFilter` 对正文图片做三级分类：

| 类型 | 判断 | 处理 |
|------|------|------|
| 水印 | `src` 以 `data:` 开头 | 删除 |
| 占位图 | URL 含 `/loading_dark.` | 删除 |
| 重复图 | 同章内相同 URL 出现多次 | 只保留首次，`exclude` 参数跳过 |
| 生僻字图 | `src` 含 `res.weread.qq.com/wrepub/`，class 含 `h-pic` | 按坐标回插到文本流中，不再重复附在文末 |
| 正常插图 | 其余 URL | 以 `![](url)` 格式写入正文文末 |

### 5. 生僻字处理（坐标回插）

详见 `docs/rare-char-rendering.md`。核心机制：

1. **采集**：`.passage-content img.h-pic`（src 含 `wrepub/`），解析 `transform: translate(x, y)` 得到 CSS 坐标。
2. **坐标统一**：canvas 内部 px ÷ scale（`wrpaHandler.getCanvasScale()`），图片 CSS px ÷ scale，统一到 canvas-CSS 空间。
3. **行内插入**：在 `getLinesWithCoords()` 返回的每行 `fragments` 中，找第一个 `xCss ≥ 图片.x` 的片段，把 `![](local_path)` 插在其前（落在 canvas 留空的间隙）。无匹配片段时追加行末。
4. **逐页独立**：翻页模式下每页 DOM 只含本页生僻字、每页 canvas y 坐标独立，因此每页单独采集 + merge，不跨页。

### 6. 翻页与断点续爬

- **翻页**：检测页脚按钮——"下一页"表示本章还有更多页，点击并提取；"下一章"表示本章结束。安全上限 200 页防止死循环。
- **停顿变量**：`--delay` 控制操作间隔，防止过快触发风控。
- **断点续爬**：每章完成后更新 `cache/{书ID}/state.json`（含 `current_chapter_index`、`completed_chapters`、`warnings`）。下次启动自动检测并续爬。

## 数据流（TOC 模式，WRPA 章节）

```
CLI --ExportRequest--> WeReadPlatform.export()
  --> WeReadCrawler.crawl()
    --> goto_reader + ensure_logged_in
    --> save_cover --> cache/{id}/封面.jpg
    --> extract_toc  --> cache/{id}/toc.json
    --> while current_index < total:
         │
         ├─ clear_wrpa_markdown()
         ├─ goto_toc_item(chapter_index)
         │
         ├─ extract_full_chapter()           ← 改造：逐页翻页
         │    ├─ Page 1: extract_chapter_content()
         │    │    ├─ wait_for_chapter_render()
         │    │    ├─ getMarkdown()  → 本页 canvas 文本
         │    │    ├─ _rare_char_images() → 本页生僻字
         │    │    ├─ _download_rare_chars() → images/wr*.png
         │    │    └─ merge_rare_chars() → 回插生僻字
         │    │
         │    ├─ while 下一页:
         │    │    ├─ clear_wrpa_markdown()
         │    │    ├─ click 下一页 / ArrowRight
         │    │    ├─ wait_for_chapter_render()
         │    │    └─ extract_chapter_content()  (同上)
         │    │
         │    └─ return "\n\n".join(各页 markdown)
         │
         └─ write content/{N}.md
           save state.json
```
