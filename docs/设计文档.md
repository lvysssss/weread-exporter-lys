处理层：
1. 预处理 (pre_process_markdown)
在将 Markdown 转换为目标格式之前，需要进行数据清洗和资源本地化：

移除水印：通过正则表达式删除 base64 编码的内嵌图片（WRPA 混淆水印）。
格式规范化：重新排版代码块与空行，确保 Markdown 语法解析正确。
图片本地化：扫描 Markdown 中的远程图片 URL，将其下载到本地 images/ 目录，并将文档中的链接替换为本地相对路径（如 images/{md5}.jpg）。
2. 后处理 (post_process_markdown)
在生成最终文件前，对 Markdown 文本进行深度清洗与结构优化：

字符清理：删除所有反引号，清理腾讯随机插入的中文汉字间换行符。
标签重组：将 <sup>[数字]文字</sup> 重排为 <sup>[数字]</sup>文字，以符合排版规范。
注释跳转链接：如果启用 footnote_link，会为上下标的注释编号生成配对的锚点（<a href="#note_x_y"> 与 <a id="note_x_y">），实现正文与注释间的双向跳转。
3. 格式转换
转 TXT (markdown_to_txt)：先将 Markdown 转为 HTML，再利用 BeautifulSoup 提取纯文本内容。
转 PDF (markdown_to_pdf)：依赖 weasyprint 库，将 HTML 与 CSS 结合渲染为 PDF。
内部 HTML 转换 (_markdown_to_html)：使用 markdown 库将 Markdown 转为 HTML 片段，启用 fenced_code 和 attr_list 扩展，并在章节末尾注入 <div class="page-break"></div> 用于分页控制。
4. 合并 (merge_markdown)
遍历所有章节，按顺序将各个章节的 Markdown 文件拼接合并为一个完整的单一 Markdown 文件。
5. 按照要求，处理成epub，mobi，pdf，azw3等
